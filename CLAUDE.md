# CLAUDE.md — 運用ルール

## 最優先ルール：データ

**本リポジトリでは架空・匿名化データのみを使用し、実在の法人名・実データは一切含めない。**

- Public リポジトリでの運用を前提とする。実データ・個人情報・秘密情報をコミットしない。
- 架空データは `src/data/*.json` に置き、生成は `scripts/gen*.ts` で行う。法人名・施設名・氏名はすべて架空とし、「（架空）」等が伝わる表記にする。
- 実データ用のファイル名パターン（`*_real.csv`、`/private-data/` など）は `.gitignore` 済み。
- 将来、実データや個人情報収集フォームを扱う場合は、その時点で保存先・送信先・セキュリティレベル・社内ポリシー適合を改めて確認してから着手する。

## プロジェクトの目的

介護福祉法人の給与担当者向け業務効率化サービス（サービス名「照合台帳」）の、商談用デモポートフォリオサイト。全体像は [IMPLEMENTATION_PLAN.md](./IMPLEMENTATION_PLAN.md) を参照。順序はポートフォリオサイト → デモ1〜4。

## 事業上の境界（重要）

このサービスが提供するのは **計算・データ処理・検算・根拠提示まで**。以下は行わない／UIでも約束しない：

- 行政への申請書作成・提出、賃金台帳など法定帳簿の代行作成、加算要件の法的判断（社会保険労務士法の独占業務に踏み込むため）。
- AIによる最終確定。**計算の確定は必ず人間（担当者）**が行う設計にする。判断できないケースは推測せず「保留（hold）」で停止する。

デモやコピーを書くときは、この境界を崩さないこと。

## 技術メモ

- スタック: Vite 8 + React 19 + TypeScript + Tailwind CSS v4（`@theme` は `src/index.css`、`tailwind.config.js` は不要）+ react-router-dom v7。
- 計算ロジックは `src/lib/ruleEngine.ts` に集約し、単体テスト（`src/lib/ruleEngine.test.ts`, Vitest）を伴わせる。各デモはこのエンジンに入力データと画面を載せ替える形で作る。
- 照合の判定は `match`（一致）/ `diff`（差異）/ `hold`（保留）の3値。色トークンとして第一級で扱う（`src/lib/status.ts`）。
- UI は `src/components` の共通部品を再利用する（デザインシステムは `/styleguide` で確認）。署名要素は「照合台帳（ReconLedger）」＝現行→再計算→差異→判定の行。
- 金額・計算値は等幅（IBM Plex Mono, `.tnum`）でタブラー表示に統一する。

## よく使うコマンド

```bash
npm run dev        # 開発サーバー
npm run build      # 型チェック + ビルド + 404.html 生成
npm test           # 単体テスト
npm run lint       # oxlint
npm run gen:demo1  # デモ1データ再生成
```

## Git 運用

- `main` は常にデプロイ可能に保つ。フェーズごとに `feature/...` ブランチを切り、PR で main にマージ（1フェーズ = 1 Issue = 1 PR を目安）。
- push 時に lint・テスト・ビルドが走り、`main` へのマージで GitHub Pages へ自動デプロイされる。
